Playtesting

Defend Together was playtested in three sessions at different developmental periods to gather feedback and actionables for the next version.

Prototype 1

This prototype was the first playable prototype we designed to demonstrate the core tower defense concept we envisioned for the final product. Contrary to other tower defense games, there exists a player character that can be controlled to attack enemies. The player is seen as a blue/red circle w/ an option for co-op gameplay in the main menu. The map is formed using simple square plots to differentiate the path from the background. Waves of enemies gradually work their way (via one path) toward the main tower, a structure intended for players to protect. The main gameplay functions are not included such as healthbars, a game-over state, defense placement.

Feedback

  • Auto targeting for player fire would be nice.
  • Implement placeable defenses.
  • Auto fire by holding the fire key.
  • Game balance — tweak HPs and player DPS.
  • Add meaning to levels.
  • Add wall collision and map boundaries.

Actionables

  • Add some function or incentive to XP and leveling to bolster progression.
  • Implement a defense placement system to support the core mechanic.
  • Add auto-fire when holding a key down to relieve finger-tap fatigue.
  • Adjust enemy stats to reduce difficulty.
  • Add map boundaries to prevent players from going off-screen.
  • Play Prototype 1

Alpha

The second iteration of our game was not uploaded correctly due to technical issues shortly before playtesting took place, so any feedback would be on an earlier version that does not reflect the latest build. However, we were still able to gather ideas for player control. The build includes game over states for player death and when the main tower is destroyed, player & tower healthbars. New enemy types were added as well, but they have not been implemented correctly and are invincible.

Feedback

  • Shooting key is too far; players had to use both hands to play.
  • Show direction of shooting on player.
  • Show health.
  • Key movement for 2nd player — may use arrow keys.
  • Add tutorials.
  • Add health bar for towers and a game over script.
  • Add shooting direction indicator for player — possibly mouse control for solo player.
  • Limit to one shooting direction; movement should not change shooting direction.

Actionables

  • Optimize controls to only require one hand per player.
  • Add visuals to indicate shooting direction on player models.
  • Include health bars on the UI for player, enemy, and main tower to show game states.
  • Add alternate controls for Player 2 (e.g., arrow keys).
  • Implement a control scheme or tutorial to assist new players.
  • Add mouse control for solo gameplay to reduce the learning curve.
  • Change the player attack method to make it feel less frustrating.
  • Play Alpha

Beta

The alpha prototype is the third iteration of our game. The game has been fleshed out to include player auto-targeting to essentially guarantee that player projectiles will hit enemies. There is now an XP bar and more enemy types that spawn at higher waves. The map layout was changed to fit the theme of the game (fantasy) and look better than previous builds that had a half-baked look to them. Holding the fire key will now auto-fire player attack, map now has a boundary, and enemies now follow random paths, spawning at random start points to prevent camping.

Feedback

  • Tower health doesn't change.
  • Bullets don't delete on new enemies.
  • Turrets should only be placed within walls.
  • Walls are free.
  • Turrets don't make money; new enemies have too much health.
  • Screen is clipping out of bounds.
  • Selected item indicator is missing.
  • No defense range indicator.
  • Items should preview after selection.
  • Make canvas auto-set resolution.
  • Layering issues present.
  • All walls around are attacked at once.
  • Control scheme needs improvement.
  • No tutorial provided.
  • Dying yellow circle is confusing.
  • Controls are confusing.

Actionables

  • Fix enemy collision with player projectiles.
  • Add cost to barricades and allow defenses to give the player XP/gold when enemies are killed.
  • Add inventory selection indicator on UI to show which item is selected.
  • Add a visible range indicator around defenses.
  • Fix all barricades being hit at once when stacked.
  • Add a tutorial to teach new players controls and mechanics.
  • Fix the confusing yellow circle (artifact from an old build).
  • Improve control scheme for better feel.
  • Balance game to make it feel fairer and more engaging.
  • Play Beta

Summary

The most common feedback we received from all the playtests was the control scheme. It seems like it was hard for new players to get used to 2-player keyboard controls, or figure out how to actually play the game, so focus should be pivoted to make controls more obvious, offering a tutorial level/screen to ease the transition for newer players.
